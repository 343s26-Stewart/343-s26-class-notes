## Web Content

In class we will use HTML, CSS, and JS to build web applications.  Each language is specialized to complete a certain part of your website.
HTML Syntax

- HTML stands for Hyper Text Markup Language, its sntax is designed with the idea of content and meta data about said content.
- For HTML syntax we have the ability to talk about single point location of content and the content needs to go there
- HTML documents are composed of CONTENT and HTML elements

### HTML

HTML Coding

- An HTML elemts is identified in the HTML document by tags
    - A tag consists of the element name within angle brackets
- The element name appeats in both the opening tag and the closing tag.  This allows us to specify a rang of things
- HTML elements can also contain attributes.  An HTML attribute is a name=value pair that provides more information about the element
- ex of html <a href = "http://www.centralpark.com"> Central Park</a>
    - the text "Central Park" is the hyperlink
    - the a is the element name
    - the href is the attribute
    - the "Central Park" is the content

#### Empty Element

- An empty element does not contain any text content; instead it is an instruction to the browser to do something
- Perhaps the most common empty element is <img>, the image element
- there is no closing img tag


#### Nesting HTML Content

- if you right click on anything and you say inspect the very specific thing you right clicked on it is going to show you teh html for that item.
- HTML tags are like first order children, everything is inside these tags
- HTML elements have parents and children.  The children are nested inside of the parent

##### Correct Nesting

- in order to properly construct this hierachry of elements your browser expects each HTML nested element to be properly nested
- a child's ending tag must occur before its parents ending tag

#### Semantic markup

- HTML documents should only focus on the structure of the document
-  HTML is SUPPOSSED to stop when we tell the computer how the content is presented.  Style is CSS's job
- So if just HTML there is still some changes in how the content is presented, but not much.  HTML will just say if something is a heading, list, etc.  NOT how the heading, list, etc should work
- Why?  For ease of developer and readability.  Also a separation of concerns and easier to make reusable and maintainable code.
