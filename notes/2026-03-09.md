## Responsive Design

* often only means having designs scale up and down with the viewport size (usually just its width), but we can also:
    * render higher resolution photos only when the device's pixel density is higher
    * choose a completely different image in different conditions, e.g. to implement "art direction"
    * design parts of our layout/presentation/aesthetic to be relative to the writing-mode

## Anchor behavior

* target="_blank" is how you instruct a link to open in a new "window"
    * "In a tabbed browser, each tab is represented by its own Window object; the global window seen by JavaScript code running within a given tab always represents the tab in which the code is running..."
        * For more, see [Window on MDN](https://developer.mozilla.org/en-US/docs/Web/API/Window)
